1.Instalação

  1. Feche o MetaTrader 5 (se estiver aberto).
  2. Abra a pasta de dados: File → Open Data Folder.
  3. Copie Dralvo_GoldMaster.ex5 para MQL5\Experts\. (Ou copie o .mq5 e clique em Compile (F7).)
  4. Fonte dos dados CFTC — escolha uma: (a) ao vivo via API (recomendado) → defina UseLiveAPI=true + coloque a URL na lista de permissões (seção 2), sem precisar de CSV; (b) por arquivo → deixe UseLiveAPI=false e é obrigatório copiar cftc_status.csv para MQL5\Files\ (faltando isso, aparece CFTC FILE NOT FOUND). O backtest sempre precisa do CSV.
  5. Reabra o MT5 → Navigator → Expert Advisors, clique com o botão direito em Refresh.
  6. Abra um gráfico de XAUUSD, mude para o timeframe D1 e arraste o Dralvo GoldMaster para o gráfico.
  7. Aba Common: marque Allow Algo Trading. Aba Inputs: informe sua License key + MT5 Account (seção 2), carregue um .set.
  8. Clique em OK → ative o Algo Trading na barra de ferramentas (fica verde).
Este é um sistema diário: os sinais são avaliados uma vez por novo candle D1. É normal passar vários dias sem operação — não espere entradas intradiárias.

2.Ativação da licença — OBRIGATÓRIA em demo/live

O EA tem uma trava de licença: em um gráfico real, ele chama o servidor Dralvo para validar a chave. Sem essa etapa o EA NÃO funciona e mostra o aviso Contact telegram: @edgardinh86.

No Strategy Tester (backtest) a trava de licença é ignorada — o backtest sempre roda sem a chave.

Passo 1 — Obtenha sua License key

Como obter a chave depende do seu plano:

PlanoOnde obter a License key
VIPFaça login no Dashboard em https://www.dralvo.com → License / Key → gere a chave para cada conta MT5.
FreeContate o Telegram @edgardinh86 para receber uma License key.
  • Em qualquer plano, a chave é vinculada a um número de conta MT5 específico — informe/registre o número correto.
  • Veja seu número de conta em: Navigator, ou Tools → Options → Server, ou no título da janela do MT5 (6–10 dígitos).

🔹 Plano VIP — sem vagas de conta: cada conta VIP recebe um número definido de vagas. Quando você usar todas (máx.) e quiser adicionar uma nova conta, contate o administrador para receber mais.

Passo 2 — Whitelist WebRequest ⚠️ Etapa mais esquecida

  1. Abra Tools → Options (Ctrl+O).
  2. Vá para a aba Expert Advisors.
  3. Marque ☑ Allow WebRequest for listed URL.
  4. Digite exatamente: https://www.dralvo.com
  5. Clique em OK.
A mesma URL também é usada pela API CFTC ao vivo quando UseLiveAPI=true. Sem essa liberação → a licença falha e o painel mostra API: OFFLINE. Este é o motivo nº 1 do EA não rodar mesmo com a chave preenchida.

3.Como este EA opera

A cada novo candle D1, o EA só abre uma compra (BUY) quando todos os filtros são atendidos ao mesmo tempo:

FiltroSignificado
CFTC bullishO posicionamento do CFTC está de lado comprado. Fonte: ao vivo + UseLiveAPI=true → direto da API; caso contrário (backtest, ou UseLiveAPI=false) → lido de cftc_status.csv.
TRENDEMA50 > EMA200 no D1 (tendência de alta).
PB (Pullback)O preço recuou o suficiente em relação à máxima recente (PullbackPct, PullbackBars).
HL (Higher-Low)O fundo do candle mais recente está acima do menor fundo de ~20 candles atrás (estrutura de alta).

Também exige um cooldown (≥ 5 dias desde a última operação, no máximo 1 por dia). Na entrada: SL = preço − 1.5×ATR, dimensionamento por % de risco. Gestão: trailing TrailATR×ATR a partir da máxima, fechamento forçado após MaxHoldDays.

O painel mostra 3 indicadores TREND / PB / HL (verde = aprovado, vermelho = ainda não). A aba Experts/Journal imprime uma linha ENTRY-CHK … a cada candle informando exatamente qual filtro bloqueou a entrada.

6.Arquivos .set & dados CFTC

4 setfiles por perfil de risco (verificados — veja a seção 7):

SetfileRisco/operaçãoPara
Dralvo_GoldMaster_SAFE.set1%Preservação de capital / contas pequenas (DD ~6%).
Dralvo_GoldMaster_BALANCED.set2%Padrão recomendado (DD ~11%).
Dralvo_GoldMaster_AGGRESSIVE.set3%Crescimento mais rápido (DD ~15%).
Dralvo_GoldMaster_MAX.set5%Lucro máximo, DD alto (~24%).
Arquivo de suporteLocalFunção
cftc_status.csvMQL5\Files\Dados diários do CFTC. Obrigatório para backtests e para live com UseLiveAPI=false. Não necessário em live com UseLiveAPI=true (usa a API).

Carregar um set: aba Inputs → Load → escolha o arquivo do seu perfil. Salvar sua própria configuração: Save.

Setfiles por tamanho de conta (seleção rápida)

Para manter o mesmo padrão dos outros EAs, também há arquivos Dralvo_GoldMaster_<PERFIL>_<tamanho>.set (24 arquivos = 4 perfis × 6 tamanhos: 1k/5k/10k/20k/50k/100k). Basta escolher o perfil + saldo corretos e carregar.

💡 Importante: o GoldMaster calcula o lote inteiramente pelo RiskPercent (% do saldo × distância do SL), então ele se ajusta automaticamente a qualquer tamanho de conta. Por isso, arquivos do mesmo perfil (ex.: todos os BALANCED_*) são idênticos — o tamanho no nome do arquivo serve só para escolha rápida e não muda o funcionamento do EA. Um único arquivo de perfil-base já serve tanto para 1.000 quanto para 100.000 USD.
Se usar o arquivo CSV (UseLiveAPI=false), ele deve ser atualizado periodicamente para que o "CFTC bullish" reflita o presente. Para evitar atualizações manuais → defina UseLiveAPI=true para o EA buscar via API. InpLicenseKey/InpMT5Account são individuais e não ficam salvos em arquivos .set compartilhados.

7.Desempenho do backtest (verificado)

Números obtidos em XAUUSD D1, dados históricos reais da Exness (modelo 1-minute OHLC), período jul/2018 → jun/2026 (~8 anos), alavancagem 1:500, sem TP fixo (saída por trailing). Período completo: 94 operações, taxa de acerto de 39,4%, Profit Factor ~2,4–2,6 — entradas seletivas, lucro alto por operação.

Importante: os resultados dependem muito da completude do histórico de dados da corretora (uma corretora com histórico de ouro incompleto gera menos operações e lucro menor). Backtests não garantem o futuro; spread/comissão/slippage reais afetam o resultado.

8.Solução de problemas

EA não inicia, log mostra CFTC FILE NOT FOUNDVocê está com UseLiveAPI=false mas não copiou cftc_status.csv para MQL5\Files\. Solução: copie o arquivo, ou defina UseLiveAPI=true (busca via API ao vivo, lembre de liberar a URL). O backtest sempre precisa do arquivo.
Sem operações por muito tempoNormal — sistema diário que precisa de CFTC + TREND + PB + HL ao mesmo tempo e ≥ 5 dias desde a última operação. Veja a linha ENTRY-CHK no Journal.
Painel API: OFFLINEVocê tem UseLiveAPI=true mas não liberou a URL / sem conexão. Defina temporariamente UseLiveAPI=false para usar o CSV.
Lote muito grande/pequenoAjuste RiskPercent.
Operação mantida por muito tempoFaz parte do design de swing; reduza MaxHoldDays ou TrailATR para sair mais cedo.

Veja detalhes nas abas Toolbox → Experts e Journal (ENTRY-CHK, ENTRY-TRY, ENTRY-OK/FAIL).

9.Aviso de risco

  • Operar com margem envolve alto risco. Nenhuma configuração garante lucro.
  • Sempre rode um demo / forward test por tempo suficiente antes de usar dinheiro real.
  • Os arquivos .set incluídos são um ponto de partida; ajuste-os à sua corretora e à volatilidade real.
  • A qualidade dos dados CFTC afeta diretamente os sinais; o backtest não garante o futuro.
  • Opere apenas com capital que você pode perder.
